DVLA Driver Points Script

Aim:

To improve driver onboarding efficiency by automating a repetitive process

Context:

For a driver to be onboarded onto my previous employer’s platform, one of the onboarding agents had to manually check to see if the driver had not committed any serious offences e.g drink driving or driving without insurance.

The process required three pieces of key information:

-Driver Licence Number

-National Insurance Number

-Postcode

After obtaining driver permission to check their points for them, we could internally query our system to pull a .csv of the above information.

The Python script is designed to run with Selenium and Chromedriver.

Script flow:

- Selenium is called and opens the appropriate DVLA website (UK Government website)

- The script pulls the first row from the saved .csv (containing Driver Licence, National Insurance, Postcode)

- The script sends the information to the associated fields on the DVLA Website

- The script confirms the necessary fields and requests to log in

- (If there is a failure due to incorrect information, the script notes this and moves on to the next row)

- Once logged in, the script copies the driver’s points information and saves this information to a .txt file

- Once the script has looped through the list it ends and the onboarding agents can then compare the results against which drivers are booked in for onboarding that day

Another benefit that came out of this project was that our onboarding agents could proactively call drivers that were not compliant, preventing them from coming in.

There was a decrease in overall driver onboarding time, an increase on the total number of onboards per day and an increase in driver satisfaction from their onboarding experience.

def post_process():
    import csv
    from os import path, remove

    bad_lines = ["You have", "To find out more", "Show information", "Start date:",
                 "Period:", "End date:", "Offence date:"
                 ]

    # Separate list needed to keep the line from deleting completely
    # e.g. '0 current penalty points' = '0 points'
    bad_words = ["current penalty "]

    # Removes unnecessary words and lines (above) and saves to another file
    with open("raw_output.txt", "r") as oldfile, open("lean_output.txt", "w") as newfile:
        for line in oldfile:
            for word in bad_words:
                line = line.replace(word, "")
            if not any(bad_line in line for bad_line in bad_lines):
                newfile.write(line)

    # Converts the previously 'leaned' output to a single line and saves it to another file
    with open("lean_output.txt", "r") as oldfile, open("singleline_output.txt", "w") as newfile:
        newfile.write(" ".join(line.strip() for line in oldfile))

    # Reorders the single line output
    with open("singleline_output.txt", "r") as oldfile, open("final_output.txt", "w") as newfile:
        reader = csv.reader(oldfile, delimiter="D", skipinitialspace=True)
        for row in reader:
            newfile.write("\n".join(row).lstrip())

    if path.exists("count.txt"):
        remove("count.txt")

    if path.exists("lean_output.txt"):
        remove("lean_output.txt")

    if path.exists("singleline_output.txt"):
        remove("singleline_output.txt")
        exit()

PCO Start Date Generator

A piece of code that predicts the start date of a document

A Private Hire Driver's Licence (PCO) is a mandetory document that is required for drivers to hold if they wish to partner with companies like Uber, Bolt and Ola in London, UK.

At my previous employer (a competitor of the above mentioned companies) we found an opportunity where we could save time autocompleting the start date of a PCO if we already knew the expiry date.

By getting the driver to upload the document and input the expiry date only, we were able to predict the start date of the document automatically with the code below.

The introducton to this script meant we could remove the start date requirement for drivers when they were uploading documents, save time for document validation agents and increase accuarcy when auditing documents internally.

In this case, the example expiry date is '13/08/2022'.

The code takes three years away from 2022, then adds one day to 13.

The result = '14/08/2019'

Code:

      
              

import datetime

expiry_date = '13/08/2022'

exp_day = datetime.datetime.strptime(expiry_date, '%d/%m/%Y').strftime('%d')

month = datetime.datetime.strptime(expiry_date, '%d/%m/%Y').strftime('%m')

year = datetime.datetime.strptime(expiry_date, '%d/%m/%Y').strftime('%Y')

start_day = str(int(exp_day) + 1)

start_month = month

start_year = str(int(year) - 3)

start_date = start_day + "/" + start_month + "/" + start_year

print("Original expiry date: " + expiry_date)
print("Predicted start date: " + start_date)
print("The predicted start date has added one day to the 'day' and deducted three from the 'year'")

Result

                             
                             
Original expiry date: 13/08/2022
Predicted start date: 14/08/2019
The predicted start date has added one day to the 'day' and deducted three from the 'year'

Generating an Annual Statement using SQL

An example of a piece of work I completed at my previous employer.

Aim:

To improve our customer support efficiency, by being able to automatically generate Annual Statements for our Taxi fleet.

The process to provide an Annual Statement took approximately five minutes per statement.

The attached code was designed to be run in Metabase, download the result as a .csv and send to the appropriate driver.


        
          SELECT bp.completion_date                 AS completion_date,
         Min(d.id_public)                   AS driver_id,
         Min(CASE bp.last_attempt_psp_type
               WHEN 'BRAINTREE' THEN 'APP PAYMENT'
               WHEN 'CREDIT' THEN 'APP PAYMENT'
               WHEN 'ADYEN' THEN 'APP PAYMENT'
               WHEN 'WIRECARD' THEN 'APP PAYMENT'
               WHEN 'CASH' THEN 'CASH'
             END)                           AS payment_method,
         SUM(bp.last_attempt_payment_value) AS gross_amount,
         MIN(CASE
               WHEN completion_date < timestamp '2019-07-31 00:00:00.000' THEN
               (bp.last_attempt_payment_value - bp.last_attempt_payment_value * 0.9) / bp.last_attempt_payment_value * 100
               WHEN completion_date > timestamp '2019-07-31 00:00:00.000' THEN
               (bp.last_attempt_payment_value - bp.last_attempt_payment_value * 0.85) / bp.last_attempt_payment_value * 100
  		   END)                           AS commission_percentage,
         SUM(CASE
               WHEN completion_date < timestamp '2019-07-31 00:00:00.000' THEN
               bp.last_attempt_payment_value * 0.9
               WHEN completion_date > timestamp '2019-07-31 00:00:00.000' THEN
               bp.last_attempt_payment_value * 0.85
             END)                           AS net_after_commission
  FROM   dm.fact_booking_payment bp
         join dm.dim_driver d
           ON bp.id_driver = d.id
  WHERE  d.id_public = '3843GM' --id_driver should be searchable
         AND bp.is_completed = TRUE
         AND bp.completion_date BETWEEN timestamp '2019-07-28 00:00:00.000' AND
                                        timestamp '2019-08-03 00:00:00.000'
  GROUP  BY rollup ( bp.completion_date )
  ORDER  BY completion_date DESC
